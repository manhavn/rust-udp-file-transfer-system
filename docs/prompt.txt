tôi muốn tạo một server rust và client app rust gồm (android app, linux cli, macos cli, window cli) có thể truyền dữ liệu gửi udp lên server như sau: mỗi block udp được gửi lên server khoảng 16kb không cố định các lần gửi, giả sử kết thúc gửi không đủ 16kb gồm: (trạng thái gửi) 255u8, (mã gói tin) 255u8, (seek begin) 255u8, (data) => nhận bao nhiêu sẽ trả về số lượng đã nhận được, cứ như thế gửi hết data thì báo (trạng thái kết thúc) 255u8, (mã gói tin) 255u8, (seek begin) 255u8 thành công và kết thúc gửi. có thể example bằng một file data. demo gửi: Trạng thái 1 đang gửi, mã gói tin 12525465011048, seek begin từ 16384 data u8 [0,1,2,3,...] => [1,255, 125,254,65,0,110,48,255, 163,84,255, 0,1,2,3,...] - Trạng thái kết thúc 2, mã gói tin 12525465011048, seek begin số byte khớp với độ dài file từ client 1020085001163 => [0,255, 125,254,65,0,110,48,255, 102,0,0,85,0,0,116,3,255] * lưu ý bắt đầu gửi thì seek begin là 0 => [1,255, 125,254,65,0,110,48,255, 0,255, 0,1,2,3,...] mã gói tin sẽ được quy định trước khi 2 bên đồng ý truyền data upload. upload xong sẽ có danh sách qua api list có trạng thái file hoàn thành hoặc đang nhận, có thời gian gửi file. mã gói tin sẽ là mã băm từ hash 256 của file lúc bắt đầu sẽ đọc mã này sau đó convert về dạng số duy nhất. ví dụ mã là 10025531402540625556 thì (mã gói tin) 255u8 sẽ là 100,25,5,31,40,254,0,62,55,56 255u8
